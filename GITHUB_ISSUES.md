## Issue 1: [Phase 1] 프로젝트 설정 및 인프라 구축
**작업 배경**
프로젝트 개발을 시작하기 위해 Git 저장소, 패키지 관리, 디렉토리 구조 등 기초 환경을 설정해야 함.

**작업 내용**
- Git 저장소 초기화 및 `.gitignore` 설정
- `npm init`을 통한 `package.json` 생성
- 개발 의존성 패키지 설치 (`jest`, `prettier`)
- `jest.config.js` 설정 (ES Modules 지원)
- 기본 디렉토리 구조 생성 (`src/core`, `src/ui`, `src/data`, `tests`)

**인수 조건**
- `npm test` 명령어 실행 시 에러 없이 실행되어야 함 (테스트 파일이 없어도 설정 오류가 없어야 함).
- 프로젝트 루트에 `src` 및 하위 폴더가 의도대로 생성되어 있어야 함.

## Issue 2: [Phase 2] 기본 데이터 모델 구현 (Player, Lineup)
**작업 배경**
시뮬레이션의 기본 단위인 선수(Player)와 라인업(Lineup) 클래스를 정의해야 함. 코어 로직이므로 TDD로 진행.

**작업 내용**
- `Player` 클래스 테스트 코드 작성 및 구현 (속성: id, 이름, 스탯 / 메서드: `getOnBaseProbability`)
- `Lineup` 클래스 테스트 코드 작성 및 구현 (유효성 검사: 9명 필수)

**인수 조건**
- `Player` 및 `Lineup` 관련 단위 테스트가 모두 통과해야 함.
- 9명이 아닌 라인업 생성 시 에러가 발생하거나 유효하지 않음을 알리는 로직이 있어야 함.

## Issue 3: [Phase 2] 게임 메카닉 구현 (BaseSystem, OutcomeGenerator)
**작업 배경**
야구 규칙의 핵심인 주루 플레이와 타격 결과를 확률에 따라 생성하는 로직이 필요함. TDD 필수.

**작업 내용**
- `BaseSystem` 테스트 및 구현: 안타 종류에 따른 주자 진루 및 득점 계산 로직
- `OutcomeGenerator` 테스트 및 구현: 선수 스탯에 기반한 가중치 랜덤 결과(안타, 아웃 등) 생성

**인수 조건**
- 만루 홈런, 병살타 등 다양한 상황에서 주자 이동과 득점이 정확히 계산되어야 함 (테스트 코드로 검증).
- 확률 생성기가 입력된 확률 분포에 맞게 결과를 반환해야 함.

## Issue 4: [Phase 2] 시뮬레이션 엔진 구현 (GameState, GameSimulator)
**작업 배경**
단일 게임을 1회부터 9회까지 시뮬레이션하는 엔진이 필요함.

**작업 내용**

- `GameSimulator` 구현: 타석 -> 이닝 -> 게임 종료까지의 전체 루프 처리
- 전체 과정에 대한 TDD 수행

**인수 조건**
- 시뮬레이터 실행 시 9이닝 종료 후 정상적으로 최종 스코어를 반환해야 함.
- 3아웃 시 이닝이 정확히 교체되어야 함.

## Issue 5: [Phase 2] 최적화 및 통계 구현
**작업 배경**
여러 번의 시뮬레이션을 통해 기대 득점과 승률을 계산하고, 최적의 라인업을 찾아야 함.

**작업 내용**
- `StatsAggregator`: 게임 결과 집계 함수 구현
- `Optimizer`: 라인업 순열 생성 및 시뮬레이션 반복 실행 로직 구현
- TDD 수행

**인수 조건**
- 주어진 라인업으로 N번 시뮬레이션 실행 후 평균 득점과 승률이 계산되어야 함.
- 최적화 실행 시 기존 라인업보다 기대 득점이 높은 라인업을 도출해야 함 (항상은 아니더라도 로직상 시도해야 함).
